=== Log Analytics output plugin

include::{include_path}/plugin_header.asciidoc[]

==== Description

A plugin for Oracle Cloud Infrastructure (OCI) Log Analytics service.
This plugin groups incoming log events from Logstash by their log group ID,
chunking them into smaller batches of maximum 2 MB size and uploads it as a ZIP payload to OCI Log Analytics.

==== Usage

This is an example of Log Analytics output plugin config:
[source,ruby]
output {
  log_analytics {
    namespace => "TENANCY_NAMESPACE"                       (required)
    profile_name => "DEFAULT"                              (optional)
    dump_zip_file => true                                  (optional)
    zip_file_location => "/tmp/"                           (optional)
    config_file_location => "/path/to/oci_config/file.txt" (optional)
    plugin_retry_on_4xx => true                            (optional)
    retry_wait_on_4xx => 1                                 (optional)
    retry_max_times_on_4xx => 5                            (optional)
    plugin_retry_on_5xx => true                            (optional)
    retry_wait_on_5xx => 1                                 (optional)
    retry_max_times_on_5xx => 5                            (optional)
  }
}

You also want to make sure to add all mandatory fields to your log events.
The mandatory fields your events must have are the following:

  * `oci_la_entity_id`
  The Entity OCID. Value type is <<string,string>>.

  * `oci_la_log_source_name`
  The corresponding Log Source name. Value type is <<string,string>>.

  * `oci_la_log_group_id`
  The log group OCID for the log event. Value type is <<string,string>>.

Optionally you can add these fields to your log events.

  * `oci_la_log_set`
  Log Set. Value type is <<string,string>>.

  * `tag`
  You can add a tag for these events. Value type is <<string,string>>.

  * `oci_la_metadata`
  Metadata used to add things like Access Control Lists, in key-value pairs. Value type is <<hash,hash>>

You can add those fields with a filter:
[source,ruby]
input {
  your_input {
    # ... other config
    type => "your_logs"
  }
}
filter{
  if [type] == "your_logs" {
    mutate {
      add_field => {
        "oci_la_entity_id" => "your_entity_ocid"
        "oci_la_log_source_name" => "Your Log Source Name"
        "oci_la_log_group_id" => "your_log_group_ocid"
        "oci_la_log_set" => "your_log_set"
        "tag" => "some_tag"
      }
    }
    ruby {
      code => '
        event.set("oci_la_metadata",
        { "Access Control List": "key:value"
        })
      '
    }
  }
}

[id="plugins-{type}s-{plugin}-namespace"]
===== `namespace` 

  * Value type is <<string,string>>
  * Default value is `nil`

Sets the mandatory OCI Log Analytics tenancy namespace used for the requests.

[id="plugins-{type}s-{plugin}-config_file_location"]
===== `config_file_location` 

  * Value type is <<string,string>>
  * Default value is `nil`

OCI config file path.

[id="plugins-{type}s-{plugin}-profile_name"]
===== `profile_name` 

  * Value type is <<string,string>>
  * Default value is `"DEFAULT"`

Name of the profile to be used from the Config File.

[id="plugins-{type}s-{plugin}-endpoint"]
===== `endpoint` 

  * Value type is <<string,string>>
  * Default value is `nil`

Set an endpoint URL for Log Analytics client operations.

[id="plugins-{type}s-{plugin}-auth_type"]
===== `auth_type` 

  * Value type is <<string,string>>
  * Default value is `"InstancePrincipal"`

Authentication type for Log Analytics client. Available options are: `"InstancePrincipal"` and `"ConfigFile"`.
If `"InstancePrincipal"` is used, the client will initialize with an Instance Principals Security Token Signer.
If `"ConfigFile"` is used, the client will initialize with the given Config File especified in <<plugins-{type}s-{plugin}-config_file_location>>.

[id="plugins-{type}s-{plugin}-oci_domain"]
===== `oci_domain` 

  * Value type is <<string,string>>
  * Default value is `nil`

Set the OCI domain for Log Analytics client

[id="plugins-{type}s-{plugin}-dump_zip_file"]
===== `dump_zip_file` 

  * Value type is <<boolean,boolean>>
  * Default value is `false`

If enabled, the plugin will dump the payload locally into the file location specified in <<plugins-{type}s-{plugin}-zip_file_location>>.

[id="plugins-{type}s-{plugin}-zip_file_location"]
===== `zip_file_location` 

  * Value type is <<string,string>>
  * Default value is `nil`

Sets the directory where the plugin will save the payload ZIP files that are generated when <<plugins-{type}s-{plugin}-dump_zip_file>> is enabled.

[id="plugins-{type}s-{plugin}-plugin_retry_on_4xx"]
===== `plugin_retry_on_4xx` 

  * Value type is <<boolean,boolean>>
  * Default value is `false`

If enabled, the plugin will retry requests for 4XX HTTP error codes.
Note: Plugin will not retry on error code: 429 - TOO MANY REQUESTS

[id="plugins-{type}s-{plugin}-plugin_retry_on_5xx"]
===== `plugin_retry_on_5xx` 

  * Value type is <<boolean,boolean>>
  * Default value is `false`

If enabled, the plugin will retry requests for 5XX HTTP error codes.

[id="plugins-{type}s-{plugin}-retry_wait_on_4xx"]
===== `retry_wait_on_4xx` 

  * Value type is <<number,number>>
  * Default value is `3`

Delay in seconds between each retry when the payload fails to upload with 4XX HTTP error codes.

[id="plugins-{type}s-{plugin}-retry_wait_on_5xx"]
===== `retry_wait_on_5xx` 

  * Value type is <<number,number>>
  * Default value is `3`

Delay in seconds between each retry when the payload fails to upload with 5XX HTTP error codes.

[id="plugins-{type}s-{plugin}-retry_max_times_on_4xx"]
===== `retry_max_times_on_4xx` 

  * Value type is <<number,number>>
  * Default value is `17`

Max amount of retries for 4XX HTTP error codes.

[id="plugins-{type}s-{plugin}-retry_max_times_on_5xx"]
===== `retry_max_times_on_5xx` 

  * Value type is <<number,number>>
  * Default value is `17`

Max amount of retries for 5XX HTTP error codes.

[id="plugins-{type}s-{plugin}-collection_source"]
===== `collection_source` 

  * Value type is <<string,string>>

Metadata property used in payload. Merely descriptive.

Proxy IP to be used. If set, Log Analytics client will send requests via the specified proxy.
